## **Módulo 3: Uso básico**

### **3.1. Iniciar y clonar repositorios**

- 3.1.1 `git init`: Crear un Nuevo Repositorio Git
- 3.1.2 `git clone`: Clonar un Repositorio Git

### **3.2. Gestión de cambios y archivos:**

- 3.2.1 `git add`: Añadir Archivos al Área de Preparación**
- 3.2.2 `git commit`: Guarda los cambios en el repositorio
- 3.2.3 `git status`:Verificar el Estado del Repositorio
- 3.2.4 `git diff`: Verificar las Diferencias entre Cambios
- 3.2.5 `git reset`: Deshacer Cambios y Modificar el Historial
- 3.2.6 `mkdir`: Crear Directorios
- 3.2.7 `touch`: Crear y Modificar Tiempos de Archivos
- 3.2.8 `mv`: Mover y Renombrar Archivos o Directorios
- 3.2.9 `rm`: Elimina archivos o directorios.

### **3.3. Exploración del historial y navegación:**

- 3.3.1 `git log`: Ver el Historial de Commits
- 3.3.2 `git checkout`: Cambiar de Rama o Restaurar Archivos
- 3.3.3 `git branch`: Gestionar Ramas en Git
- 3.3.4 `git merge`: Fusionar Ramas en Git
- 3.3.5 `git rebase`: Reorganizar el Historial de Commits
- 3.3.6 `cd`: Cambiar el Directorio de Trabajo
- 3.3.7 `ls`: Listar Contenidos de un Directorio
- 3.3.8 `pwd`: Imprimir el Directorio de Trabajo Actual

### **3.4. Trabajo con repositorios remotos:**

- 3.4.1 `git remote add`: Añadir un Repositorio Remoto
- 3.4.2 `git remote -v`: Verificar Repositorios Remotos
- 3.4.3 `git push`: Enviar Cambios al Repositorio Remoto
- 3.4.4 `git pull`: Obtener y Fusionar Cambios desde el Repositorio Remoto
- 3.4.5 `git fetch`: Obtiene cambios desde un repositorio remoto, pero no los fusiona automáticamente.
- 3.4.6 `git remote rm`: Elimina un repositorio remoto de la configuración.

### **3.5. Funcionalidades avanzadas:**

- 3.5.1 `git cherry-pick`: Aplica un commit específico a la rama actual.
- 3.5.2 `git revert`: Deshace un commit específico creando un nuevo commit de reversión.
- 3.5.3 `git stash`: Guarda cambios locales temporalmente para trabajar en otra cosa.
- 3.5.4 `git tag`: Crea, lista o elimina etiquetas (versiones específicas) en el repositorio.
